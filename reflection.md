# Reflection

This assignment made me think much more seriously about the difference between “a chatbot that works” and “a chatbot that feels intentional.” In the beginning, I thought the main effort would go into wiring the frontend to the backend and making sure the API call worked correctly. That part mattered, but it quickly became obvious that the actual quality of the project depended much more on the prompt design. If the prompt was vague, the model answered in a vague and forgettable way. If the prompt was specific, grounded, and constrained, the responses became much more consistent and useful.

One thing that worked well was treating each persona as a separate product decision instead of using one common template with three different names. I tried to give each mentor a distinct teaching style, pace, and structure of response. Anshuman’s prompt was written to sound clear and first-principles driven, Abhimanyu’s prompt was shaped to feel sharper and more interview-oriented, and Kshitij’s prompt was made more diagnostic and patient. That separation made the frontend experience stronger too, because the persona switcher actually changes the feel of the conversation instead of only changing the title on the screen.

Another decision that helped was resetting the chat when the persona changes. It keeps the flow clean and avoids mixing contexts between mentors. On the implementation side, the backend stayed simple: the frontend only sends the selected persona, the user message, and the chat history. That made debugging easier and also kept the app more reliable. On the frontend, smaller details like suggestion chips, a typing indicator, and clear error states helped the experience feel complete without making the interface too complicated.

The biggest lesson from this assignment was how real the GIGO principle is. Better input really does lead to better output. Early prompt drafts were serviceable, but they did not produce strong persona-specific answers. Once I added clearer persona descriptions, few-shot examples, response constraints, and explicit “do not” rules, the quality improved noticeably. If I were continuing this project, I would spend even more time researching how each person actually speaks in public and refine the prompts until the difference in voice feels even more authentic.
